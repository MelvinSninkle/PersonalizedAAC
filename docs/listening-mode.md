# Listening mode (native iOS)

Speak near the iPad and the child's world renders live as a one-tile-high strip
across the header — the same greedy-longest matching as Message the Board. Started
from the board's own **🎙️ mic button** (child-accessible, not behind edit mode) or
remotely from the parent app. Auto-stops after **2 minutes** with no speech.

> This is the **native `kid-ios` app** — the board actually on the iPad. (There's
> also a web version in `app.html`, but that runtime was retired for the device.)

## How it's built (native)

- **`Audio/SpeechListener.swift`** — wraps Apple `SFSpeechRecognizer` + `AVAudioEngine`,
  streaming the live partial transcript. Uses **on-device / offline** recognition
  (`requiresOnDeviceRecognition`) when the language pack is present; restarts itself
  when iOS ends a session on silence. No plugin, no patch — native gets the Speech
  framework directly.
- **`Views/ListenStripView.swift`** — `ListenTokenizer` (greedy-longest, mirrors
  `api/message-to-board.js`) builds a lexicon from `BoardStore.tiles` and turns the
  transcript into tiles + text chips; the strip auto-scrolls to the newest word and
  tile chips are tappable to speak.
- **`Views/HeaderBar.swift`** — adds the 🎙️ mic button (left, opposite Play; red
  while listening) and swaps the branded title for the strip while active.
- **`Views/BoardView.swift`** — owns the `SpeechListener` + `listening` state, starts/
  stops the mic, enforces the 2-minute idle timeout (reset on each recognized phrase),
  and handles the remote `listen-start` / `listen-stop` live commands.

The parent trigger is the same `/api/live` command the web parent used
(`{kind:'cmd', action:'listen-start'|'listen-stop'}`). The native parent app can send
it the same way it sends other live commands.

## Setup (one-time)

`kid-ios` is generated by **XcodeGen** from `project.yml`, so there's no manual
Info.plist editing.

1. The speech permission string is already added to `project.yml`
   (`NSSpeechRecognitionUsageDescription`); `NSMicrophoneUsageDescription` was already
   there. Nothing to type.
2. Regenerate + build:
   ```bash
   cd kid-ios
   xcodegen generate      # rewrites MyWorld.xcodeproj from project.yml
   open MyWorld.xcodeproj  # or: xed .
   ```
   In Xcode: pick the iPad → set your Team under Signing & Capabilities → ▶ Run (or
   Archive → TestFlight, as usual).
3. First time listening mode starts on the iPad, iOS asks for **microphone** and
   **speech recognition** once — tap **Allow**. Never asks again.

Offline needs iOS 13+ and the language pack downloaded on the device (Settings →
General → Keyboard / Dictation). If a device lacks on-device support, Apple falls
back to online automatically.

## Try it

Unlock isn't required. Tap the **🎙️** button in the header (it turns red / becomes a
■ stop). Speak — words that exist on the board appear as tiles, others as text chips,
and the strip scrolls to the newest. Tap **■** (or wait 2 minutes) to stop; the mic
indicator turns off and the branding returns.

## Note on the web version

The web board (`app.html`) has its own listening implementation behind the Capacitor
speech plugin. It's unused on the retired web runtime but left in place; if the web
board is ever revived, its setup is in this file's git history.
